# 🌐 WordPress 연동 가이드

> **이 가이드를 처음부터 끝까지 따라하면 누구나 WordPress 자동 발행 설정을 완료할 수 있습니다.**

---

## 📖 전체 흐름 요약

```
1단계: Gemini API Key 발급 (AI 글 생성용)
   ↓
2단계: WordPress Application Password 발급
   ↓
3단계: 프로그램에 정보 입력 + 테스트
```

> 💡 블로그스팟(Blogger)에 비해 훨씬 간단합니다! Google Cloud Console 설정이 필요 없습니다.

---

## 📌 필요한 정보 한눈에 보기

| 항목 | 찾는 방법 | 예시 | 프로그램 입력 필드 |
|------|-----------|------|-------------------|
| **Gemini API Key** | [Google AI Studio](https://aistudio.google.com/apikey)에서 API 키 생성 (1단계) | `AIzaSy...xxxxx` | 환경설정 → Gemini API |
| **Site URL** | 내 워드프레스 사이트의 주소 (브라우저 주소창) | `https://myblog.com` | 환경설정 → Site URL |
| **Username** | 워드프레스 관리자 로그인 아이디 (관리자 → 사용자 → 프로필 → "사용자명") | `admin` | 환경설정 → Username |
| **Application Password** | 워드프레스 관리자 → 사용자 → 프로필 → 하단 "애플리케이션 비밀번호"에서 발급 (2단계) | `xxxx xxxx xxxx xxxx xxxx xxxx` | 환경설정 → Application Password |

> ⚠️ **Application Password ≠ 로그인 비밀번호**  
> 워드프레스에 로그인할 때 쓰는 비밀번호가 아닙니다! 프로필 페이지에서 **별도로 발급**받는 비밀번호입니다.

---

## 1단계: Gemini API Key 발급

AI가 블로그 글을 생성하려면 Gemini API Key가 필요합니다.

### 진행 방법

1. **Google AI Studio** 접속  
   👉 [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey)

2. Google 계정으로 **로그인**

3. **[API 키 만들기]** 또는 **[Create API Key]** 버튼 클릭

4. 프로젝트를 선택하라는 창이 나오면 → **기존 프로젝트 아무거나 선택** 또는 **새 프로젝트 생성**

5. 생성된 API Key를 **복사** (예: `AIzaSy...xxxxx`)

6. 프로그램의 **환경설정 탭** → **Gemini API** 필드에 붙여넣기

> ⚠️ **이미지 생성까지 하려면?**  
> Google AI Studio에서 '유료 티어'(결제 수단 등록)로 전환해야 이미지 생성 API가 활성화됩니다.  
> 글 생성만 할 경우에는 무료 티어로도 충분합니다.

---

## 2단계: WordPress Application Password 발급

WordPress REST API로 글을 자동 발행하려면 **Application Password**(애플리케이션 비밀번호)가 필요합니다.  
이것은 보안을 위해 로그인 비밀번호 대신 사용하는 **별도의 비밀번호**입니다.

### 사전 확인사항

> 🚨 **아래 조건을 모두 충족해야 Application Password 기능을 사용할 수 있습니다:**
> - WordPress **5.6 이상** (2020년 12월 이후 버전)
> - 사이트가 **HTTPS**로 운영 중 (`http://`로 시작하면 안 됨!)
> - **관리자(Administrator)** 권한의 계정

### 진행 방법 (상세 경로 포함)

1. 워드프레스 **관리자 페이지** 접속  
   👉 브라우저에 `https://내사이트주소/wp-admin` 입력 후 로그인

2. 로그인 후 **관리자 대시보드** 화면이 나타납니다

3. **왼쪽 사이드바 메뉴**에서 경로를 따라가세요:

   ```
   왼쪽 사이드바
     ├── 알림판 (대시보드)
     ├── 글
     ├── 미디어
     ├── 페이지
     ├── 댓글
     ├── 외모
     ├── 플러그인
     ├── 👉 사용자          ← 여기에 마우스를 올리세요!
     │     ├── 모든 사용자
     │     └── 👉 프로필    ← 이것을 클릭!
     ├── 도구
     └── 설정
   ```

   > 💡 **영문 WordPress인 경우**: `Users` → `Profile` 순서입니다

4. **프로필** 페이지가 열리면 **페이지 맨 아래**로 스크롤하세요  
   (이름, 닉네임, 이메일 등의 설정을 지나서 계속 내려가야 합니다)

5. 아래쪽에 **"애플리케이션 비밀번호"** (Application Passwords) 섹션이 보입니다:

   ```
   ┌─────────────────────────────────────────────────────────┐
   │  애플리케이션 비밀번호                                     │
   │                                                         │
   │  새 애플리케이션 비밀번호 이름: [________________]          │
   │                                                         │
   │  [새 애플리케이션 비밀번호 추가]                            │
   └─────────────────────────────────────────────────────────┘
   ```

   > ⚠️ **이 섹션이 안 보인다면?** → 아래 "자주 겪는 문제" 섹션의 첫 번째 항목 참고

6. **"새 애플리케이션 비밀번호 이름"** 입력칸에 이름을 입력하세요  
   (예: `블로그자동화` — 나중에 구분하기 위한 이름일 뿐, 아무거나 상관없습니다)

7. **[새 애플리케이션 비밀번호 추가]** 버튼 클릭  
   (영문: **[Add New Application Password]**)

8. 🎉 **비밀번호가 생성됩니다!** 아래와 같은 형태입니다:

   ```
   새 비밀번호:  abcd efgh ijkl mnop qrst uvwx
   ```

   > 🚨 **매우 중요: 이 비밀번호는 이 화면에서 딱 한 번만 볼 수 있습니다!**  
   > 반드시 **메모장에 복사**해두세요. 화면을 벗어나면 다시 볼 수 없습니다.  
   > (분실 시에는 기존 비밀번호를 삭제하고 새로 발급하면 됩니다)

9. 비밀번호를 복사할 때 **공백(띄어쓰기) 포함**하여 그대로 전체 복사하세요

### Username(사용자명) 확인 방법

현재 보고 있는 **프로필 페이지 상단**에 아래 항목이 있습니다:

```
사용자명:  admin       (변경 불가)
```

이 "사용자명" 값이 프로그램에 입력할 **Username**입니다.  
⚠️ **"이름"이나 "닉네임"이 아닌 "사용자명"**을 사용하세요!

---

## 3단계: 프로그램에 정보 입력 + 테스트

지금까지 준비한 정보를 프로그램에 입력하는 마지막 단계입니다.

### 진행 방법

1. 프로그램 실행 → **환경설정** 탭 이동

2. 플랫폼 선택: **WordPress** 라디오 버튼 클릭  
   (파란색 WordPress 설정 영역이 나타납니다)

3. 아래 3개 필드에 정보 입력:

   | 필드명 | 입력 값 | 예시 | 주의사항 |
   |--------|---------|------|---------|
   | **Site URL** | 워드프레스 사이트 주소 | `https://myblog.com` | `https://`로 시작, 끝에 `/` 없이 |
   | **Username** | 프로필 페이지의 "사용자명" | `admin` | 이메일 ❌, 닉네임 ❌, **사용자명** ✅ |
   | **Application Password** | 2단계에서 발급한 비밀번호 | `abcd efgh ijkl mnop` | 로그인 비밀번호 ❌, **앱 비밀번호** ✅ |

4. **[💾 설정 저장]** 클릭

---

## ✅ 설정 완료 후 테스트

1. **포스팅 탭** 이동
2. 키워드 입력 (예: `워드프레스 블로그 시작하기`)
3. **[🚀 블로그 글 작성 시작]** 클릭
4. 로그에 "✅ 발행 완료" 메시지 확인
5. 워드프레스 관리자 → 왼쪽 사이드바 **[글]** → **[모든 글]** 에서 실제로 글이 올라갔는지 확인

## 🚨 자주 겪는 문제 & 해결법

---

### ❌ 문제 1: "애플리케이션 비밀번호" 섹션이 프로필 페이지에 안 보여요

**증상**: 워드프레스 관리자 → 사용자 → 프로필 페이지를 아무리 스크롤해도 "애플리케이션 비밀번호" 항목이 없음

**원인과 해결 (3가지 중 하나):**

| 원인 | 확인 방법 | 해결 방법 |
|------|-----------|-----------|
| **WordPress 버전이 5.6 미만** | 관리자 → 왼쪽 사이드바 **[알림판]** → 우측 하단 "WordPress 버전" 확인 | 관리자 → **[알림판]** → **[업데이트]** 에서 최신 버전으로 업데이트 |
| **사이트가 HTTPS가 아님** | 브라우저 주소창이 `http://`로 시작하는지 확인 | 호스팅 업체에 SSL 인증서 설치 요청 (대부분 무료 Let's Encrypt 제공) |
| **보안 플러그인이 차단** | Wordfence, iThemes Security 등의 보안 플러그인 사용 중 | 해당 플러그인 설정에서 "Application Passwords" 또는 "REST API" 기능을 **허용**으로 변경 |

---

### ❌ 문제 2: "401 Unauthorized" 오류

**증상**: 발행 시도 시 로그에 `401 Unauthorized` 또는 "인증 실패" 메시지가 나타남

**의미**: 서버가 "너 누군지 모르겠다"고 거부하는 것. **Username 또는 Application Password가 틀렸다**는 뜻입니다.

**확인 & 해결:**

| 확인 항목 | 올바른 값 | 흔한 실수 |
|-----------|-----------|-----------|
| **Username** | 프로필 페이지의 "사용자명" (예: `admin`) | ❌ 이메일 주소 입력, ❌ 표시 이름(닉네임) 입력 |
| **Application Password** | 프로필에서 발급한 앱 비밀번호 (예: `abcd efgh ijkl`) | ❌ 로그인 비밀번호 입력, ❌ 비밀번호 앞뒤 공백 포함 |

**해결 방법:**
1. 워드프레스 관리자 → **[사용자]** → **[프로필]** → 상단 "사용자명" 확인
2. Application Password를 **새로 발급**하여 다시 입력
3. 프로그램에서 **[💾 설정 저장]** 다시 클릭

---

### ❌ 문제 3: "403 Forbidden" 오류

**증상**: 발행 시도 시 로그에 `403 Forbidden` 또는 "접근 거부" 메시지가 나타남

**의미**: 서버가 "누군지는 알겠는데, 이 작업은 허용하지 않겠다"고 차단하는 것. **서버 보안 설정이 REST API를 막고 있다**는 뜻입니다.

**확인 & 해결:**

| 순서 | 확인 항목 | 해결 방법 |
|------|-----------|-----------|
| 1 | **보안 플러그인** (Wordfence, Sucuri 등) | 플러그인 설정 → "REST API" 또는 "외부 접근" → **허용**으로 변경 |
| 2 | **호스팅 방화벽 (WAF)** | 호스팅 관리 패널(cPanel 등) → 방화벽 설정 → REST API 엔드포인트(`/wp-json/`) **허용** |
| 3 | **.htaccess 파일** | FTP로 접속 → 루트의 `.htaccess` 파일에서 `/wp-json/` 차단 규칙이 있다면 제거 |

---

### ❌ 문제 4: "404 Not Found" 오류

**증상**: 발행 시도 시 로그에 `404 Not Found` 메시지가 나타남

**의미**: 서버가 "그런 주소는 없다"고 응답하는 것. **Site URL이 잘못되었거나, WordPress의 REST API 경로가 비활성화**되어 있다는 뜻입니다.

**확인 & 해결:**

| 순서 | 확인 항목 | 해결 방법 |
|------|-----------|-----------|
| 1 | **Site URL 정확성** | `https://`로 시작하는지, 오타는 없는지, 끝에 `/`가 없는지 확인 |
| 2 | **고유주소(퍼머링크) 설정** | 관리자 → **[설정]** → **[고유주소]** → **"기본"이 아닌 다른 옵션** 선택 (예: "글 이름") → [변경사항 저장] 클릭 |
| 3 | **REST API 접근 테스트** | 브라우저에서 `https://내사이트/wp-json/wp/v2/posts` 접속 → JSON이 나오면 정상, 404면 REST API 비활성화 상태 |

> 💡 **고유주소가 "기본"으로 되어 있으면** WordPress REST API가 작동하지 않습니다.  
> 반드시 "글 이름", "날짜와 이름", "숫자" 중 하나로 변경하세요.

---

### ❌ 문제 5: "rest_cannot_create" 오류

**증상**: 발행 시도 시 로그에 `rest_cannot_create` 또는 "글 작성 권한이 없습니다" 메시지가 나타남

**의미**: 로그인한 사용자 계정에 **글을 작성할 권한이 없다**는 뜻입니다. WordPress는 사용자 역할에 따라 권한이 다릅니다.

**WordPress 사용자 역할별 권한:**

| 역할 | 글 작성 | 글 발행 | 자동 발행 가능? |
|------|---------|---------|----------------|
| 관리자 (Administrator) | ✅ | ✅ | ✅ 가능 |
| 편집자 (Editor) | ✅ | ✅ | ✅ 가능 |
| 글쓴이 (Author) | ✅ | ✅ (본인만) | ⚠️ 제한적 |
| 기여자 (Contributor) | ✅ | ❌ | ❌ 불가 |
| 구독자 (Subscriber) | ❌ | ❌ | ❌ 불가 |

**해결 방법:**
1. 관리자 계정으로 로그인
2. 왼쪽 사이드바 **[사용자]** → **[모든 사용자]** 클릭
3. 자동 발행에 사용할 사용자 이름 클릭
4. **"역할"** 드롭다운에서 **"관리자"** 또는 **"편집자"** 선택
5. **[사용자 업데이트]** 클릭

---

### ❌ 문제 6: Application Password를 잊어버렸어요

**증상**: 이전에 발급받은 Application Password를 어디에도 저장하지 않아서 모름

**해결 방법** (기존 삭제 후 재발급):
1. 워드프레스 관리자 → 왼쪽 사이드바 **[사용자]** → **[프로필]** 클릭
2. 페이지 하단 "애플리케이션 비밀번호" 섹션으로 스크롤
3. 기존 비밀번호 옆의 **[해지]** 버튼 클릭 → 삭제
4. 새 이름 입력 → **[새 애플리케이션 비밀번호 추가]** 클릭
5. 새 비밀번호를 **반드시 복사**하여 프로그램에 다시 입력
