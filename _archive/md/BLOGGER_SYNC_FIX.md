# 블로그 스팟 동기화 문제 해결 가이드

## 🔍 문제 분석 결과

블로그 발행이 실패하는 근본적인 원인을 분석한 결과, 다음과 같은 문제들을 발견했습니다:

### 1. 주요 원인
- **OAuth 토큰 파일 누락**: `token.json` 파일이 존재하지 않아 Google 인증이 실패
- **인증 플로우 개선 필요**: 기존 인증 과정에서 사용자 안내가 부족하고 오류 처리가 미흡
- **환경 변수는 정상**: Google API 키들이 모두 올바르게 설정되어 있음

### 2. 해결된 사항
- ✅ OAuth 인증 플로우 개선
- ✅ 사용자 친화적인 오류 메시지 추가
- ✅ 상세한 디버깅 로그 추가
- ✅ 타임아웃 및 오류 처리 강화

## 🛠️ 수정된 기능들

### 1. 향상된 OAuth 인증 (`ensureAuth` 함수)
```typescript
// 개선된 토큰 확인 및 디버깅 정보
console.log('[ensureAuth] 토큰 파일 경로:', TOKEN_PATH);
console.log('[ensureAuth] 토큰 파일 존재 여부:', fs.existsSync(TOKEN_PATH));
```

### 2. 사용자 친화적인 인증 플로우 (`doAuthFlow` 함수)
- 명확한 단계별 안내 메시지
- 브라우저에서 열어야 할 URL 표시
- 인증 성공/실패 시각적 피드백
- 5분 타임아웃 설정
- 상세한 오류 처리

### 3. 강화된 게시 함수 (`postToBlogger` 함수)
- 환경 변수 검증
- 구체적인 오류 코드별 해결 방법 안내
- 상세한 진행 상황 로그
- 성공 시 게시물 정보 표시

## 🚀 사용 방법

### 1. 프로그램 실행
```bash
npm start
```

### 2. 첫 번째 실행 시 (토큰이 없는 경우)
1. 프로그램이 OAuth 인증 URL을 표시합니다
2. 브라우저에서 해당 URL을 열어주세요
3. Google 계정으로 로그인하고 권한을 승인해주세요
4. 인증 완료 후 자동으로 `token.json` 파일이 생성됩니다

### 3. 이후 실행 시
- 토큰이 자동으로 로드되어 바로 사용 가능합니다
- 토큰이 만료된 경우 자동으로 재인증을 요청합니다

## 🔧 문제 해결

### 인증 오류 (401)
```
[postToBlogger] 인증 오류: OAuth 토큰이 유효하지 않습니다.
해결 방법: 프로그램을 다시 시작하여 재인증하세요.
```

### 권한 오류 (403)
```
[postToBlogger] 권한 오류: Blogger API 접근 권한이 없습니다.
해결 방법: Google Cloud Console에서 Blogger API를 활성화하세요.
```

### 블로그를 찾을 수 없음 (404)
```
[postToBlogger] 블로그를 찾을 수 없습니다.
해결 방법: BLOG_ID가 올바른지 확인하세요.
```

### 잘못된 요청 (400)
```
[postToBlogger] 잘못된 요청: 콘텐츠에 문제가 있을 수 있습니다.
해결 방법: 콘텐츠를 확인하고 다시 시도하세요.
```

## 📋 환경 변수 확인

다음 명령어로 환경 변수가 올바르게 설정되었는지 확인할 수 있습니다:

```bash
node check-env-keys.js
```

필수 환경 변수:
- `GOOGLE_CLIENT_ID`: Google OAuth 클라이언트 ID
- `GOOGLE_CLIENT_SECRET`: Google OAuth 클라이언트 Secret
- `GOOGLE_BLOG_ID`: 블로그 ID
- `GOOGLE_REDIRECT_URI`: 리디렉션 URI (기본값: http://localhost:8080)

## 🎯 주요 개선사항

1. **명확한 사용자 안내**: 각 단계별로 무엇을 해야 하는지 명확히 안내
2. **상세한 오류 정보**: 문제 발생 시 구체적인 해결 방법 제시
3. **자동 토큰 관리**: 토큰 만료 시 자동으로 재인증 요청
4. **타임아웃 처리**: 인증 과정에서 무한 대기 방지
5. **시각적 피드백**: 브라우저에서 인증 상태를 명확히 표시

## ✅ 테스트 방법

1. 프로그램을 실행합니다
2. 블로그 발행을 시도합니다
3. 첫 실행 시 OAuth 인증 과정을 확인합니다
4. 인증 완료 후 정상적으로 게시되는지 확인합니다
