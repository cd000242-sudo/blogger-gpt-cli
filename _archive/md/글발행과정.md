# 글 발행 과정 설명

## 전체 흐름도

```
1. 사용자가 "글 생성" 버튼 클릭
   ↓
2. runPosting() 함수 실행 (electron/ui/script.js)
   - 사용자 입력값 수집 (주제, 키워드, 설정 등)
   - API 키 검증
   ↓
3. runPost() 함수 호출 (src/core/index.ts)
   - 콘텐츠 생성 모드 결정 (MAX 모드 또는 기본 모드)
   ↓
4. generateMaxModeArticle() 실행 (MAX 모드인 경우)
   - 섹션별로 콘텐츠 생성
   - 각 섹션마다 Gemini API 호출
   ↓
5. HTML 콘텐츠 생성 및 스킨 적용
   - 프리미엄 스킨 CSS 적용
   - 목차 생성
   - 썸네일 이미지 생성
   ↓
6. publishToBlogger() 함수 실행 (src/core/blogger-publisher.js)
   - Blogger API 인증
   - 포스트 업로드
   ↓
7. 발행 완료 및 URL 반환
```

## 상세 단계별 설명

### 1단계: 사용자 입력 수집
**파일**: `electron/ui/script.js` - `runPosting()` 함수

```javascript
// 수집되는 정보:
- 주제 (topic)
- 키워드 (keywords)
- 최소 글자수 (minChars)
- 콘텐츠 모드 (contentMode)
- 프롬프트 모드 (promptMode)
- 썸네일 설정
- 발행 모드 (draft/publish/schedule)
- CTA 설정
```

### 2단계: 콘텐츠 생성 모드 결정
**파일**: `src/core/index.ts` - `runPost()` 함수

```typescript
// promptMode가 'max-mode'인 경우
if (promptMode === 'max-mode') {
  // MAX 모드로 콘텐츠 생성
  const result = await generateMaxModeArticle(maxModePayload, env, onLog);
  html = result.html;
  title = result.title;
}
```

### 3단계: MAX 모드 콘텐츠 생성 (핵심)
**파일**: `src/core/index.ts` - `generateMaxModeArticle()` 함수

#### 3-1. 섹션 구조 정의
```typescript
// 기본 섹션들:
- 소개 (Introduction)
- 주요 특징 (Features)
- 활용 방법 (Usage)
- 주의사항 (Considerations)
- 미래 전망 (Future)
```

#### 3-2. 각 섹션별 콘텐츠 생성
```typescript
for (const section of sections) {
  // 1. 프롬프트 생성
  const prompt = buildMaxModeSectionPrompt(
    section,
    topic,
    keywords,
    crawledContent
  );
  
  // 2. Gemini API 호출
  const sectionContent = await safeGenerateContent(
    geminiModel,
    prompt,
    {
      maxOutputTokens: 12000,
      temperature: 0.7
    }
  );
  
  // 3. HTML로 변환
  const sectionHtml = convertToHtml(sectionContent);
  
  // 4. 섹션 이미지 생성 (선택)
  if (sectionImageMode === 'dalle') {
    const imageUrl = await generateH2Image(section.title, 'dalle', payload);
  }
}
```

#### 3-3. 프롬프트 구조
각 섹션마다 다음과 같은 프롬프트가 생성됩니다:

```
당신은 한국어 SEO 전문 에디터입니다.

주제: {topic}
키워드: {keywords}
섹션: {section.title}

요구사항:
- 최소 {minChars}자 이상
- {section.description}
- {section.contentFocus}에 집중
- {section.requiredElements} 포함

크롤링된 참고 자료:
{crawledContent}

위 자료를 참고하되 절대 복사하지 말고, 
독창적이고 전문적인 콘텐츠를 생성하세요.
```

### 4단계: HTML 래핑 및 스킨 적용
**파일**: `src/core/index.ts` - `generateMaxModeArticle()` 함수

```typescript
// 1. 모든 섹션 HTML 결합
let fullHtml = sections.map(s => s.html).join('\n');

// 2. 목차 생성
const tocHtml = generateTableOfContents(sections);

// 3. 프리미엄 스킨 CSS 적용
const premiumCSS = `
  .premium-section { ... }
  .premium-body h2 { ... }
  .premium-body h3 { ... }
  ...
`;

// 4. 최종 HTML 조립
const finalHtml = `
  <style>${premiumCSS}</style>
  <div class="premium-article">
    <h1 class="premium-section-title">${title}</h1>
    ${tocHtml}
    ${fullHtml}
    ${ctaSection}
  </div>
`;
```

### 5단계: 썸네일 이미지 생성
**파일**: `src/core/index.ts` - `generateH2Image()` 함수

```typescript
// 썸네일 모드에 따라 생성:
if (thumbnailMode === 'text') {
  // SVG 기반 텍스트 썸네일
  thumbnailUrl = generateTextThumbnail(title, topic);
} else if (thumbnailMode === 'dalle') {
  // DALL-E API로 이미지 생성
  thumbnailUrl = await generateDalleImage(title, dalleApiKey);
} else if (thumbnailMode === 'pexels') {
  // Pexels API로 이미지 검색
  thumbnailUrl = await searchPexelsImage(title, pexelsApiKey);
}
```

### 6단계: Blogger 발행
**파일**: `src/core/blogger-publisher.js` - `publishToBlogger()` 함수

```javascript
// 1. Blogger API 인증 확인
const authStatus = await checkBloggerAuthStatus();
if (!authStatus.isAuthenticated) {
  // OAuth 인증 진행
  await authenticateBlogger();
}

// 2. 포스트 데이터 준비
const postData = {
  title: title,
  content: html,
  labels: labels || [],
  published: postingStatus === 'publish' ? new Date().toISOString() : null,
  updated: new Date().toISOString()
};

// 3. 썸네일이 있으면 추가
if (thumbnailUrl) {
  // 썸네일 이미지를 Blogger에 업로드
  const uploadedThumbnail = await uploadThumbnailToBlogger(thumbnailUrl);
  postData.images = [uploadedThumbnail];
}

// 4. Blogger API로 포스트 발행
const response = await fetch(
  `https://www.googleapis.com/blogger/v3/blogs/${blogId}/posts`,
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(postData)
  }
);

// 5. 발행 결과 반환
const result = await response.json();
return {
  ok: true,
  url: result.url,
  id: result.id
};
```

## 주요 특징

### 1. 섹션별 독립 생성
- 각 섹션은 독립적으로 Gemini API를 호출하여 생성
- 섹션별로 다른 프롬프트와 요구사항 적용 가능

### 2. 크롤링 콘텐츠 활용
- Google Search API로 관련 콘텐츠 크롤링
- 크롤링된 내용을 참고하여 더 전문적인 콘텐츠 생성
- 절대 복사하지 않고 독창적으로 재작성

### 3. 프리미엄 스킨
- 화려한 그라디언트 스타일
- 반응형 디자인
- H2/H3 제목에 특별한 스타일 적용

### 4. 다중 이미지 지원
- 섹션별 이미지 (DALL-E, Pexels 등)
- 썸네일 이미지
- 자동 이미지 최적화

### 5. CTA 자동 삽입
- 자동 CTA 생성 또는 수동 CTA 설정
- 섹션별로 적절한 위치에 CTA 삽입

## 에러 처리

각 단계마다 try-catch로 에러를 처리하며, 실패 시:
1. 사용자에게 에러 메시지 표시
2. 로그에 상세한 에러 정보 기록
3. 부분적으로 생성된 콘텐츠는 저장하여 복구 가능

## 성능 최적화

1. **모델 캐싱**: Gemini 모델 인스턴스를 캐시하여 재사용
2. **병렬 처리**: 가능한 경우 여러 섹션을 병렬로 생성
3. **재시도 로직**: 네트워크 오류 시 자동 재시도 (최대 3회)
4. **API 호출 최적화**: 불필요한 API 호출 최소화
