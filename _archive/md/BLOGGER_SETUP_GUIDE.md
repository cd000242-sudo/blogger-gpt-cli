# 🅱️ Blogger(블로그스팟) 연동 가이드

> **이 가이드를 처음부터 끝까지 따라하면 누구나 블로그스팟 자동 발행 설정을 완료할 수 있습니다.**

---

## 📖 전체 흐름 요약

```
1단계: Gemini API Key 발급 (AI 글 생성용)
   ↓
2단계: Google Cloud Console 프로젝트 생성
   ↓
3단계: Blogger API 활성화
   ↓
4단계: OAuth 동의 화면 설정 + 테스트 사용자 등록
   ↓
5단계: OAuth 클라이언트 ID & Secret 발급
   ↓
6단계: Blog ID 확인
   ↓
7단계: 프로그램에 정보 입력 + 인증 완료
```

---

## 📌 필요한 정보 한눈에 보기

설정에 필요한 모든 항목을 한 표에 정리했습니다. 아래 단계를 따라가며 하나씩 채워보세요.

| 항목 | 찾는 방법 | 예시 | 프로그램 입력 필드 |
|------|-----------|------|-------------------|
| **Gemini API Key** | [Google AI Studio](https://aistudio.google.com/apikey)에서 API 키 생성 (1단계) | `AIzaSy...xxxxx` | 환경설정 → Gemini API |
| **블로그 ID** | [Blogger 관리자 페이지](https://www.blogger.com) 접속 → URL의 `posts/` 뒤 숫자 부분 (6단계) | `1234567890123456789` | 환경설정 → Blog ID |
| **블로그 URL** | 내 블로그의 실제 주소 | `https://example.blogspot.com` | (자동 인식) |
| **Google Client ID** | Google Cloud Console → 사용자 인증 정보에서 OAuth 클라이언트 생성 시 발급 (5단계) | `123456-xxx.apps.googleusercontent.com` | 환경설정 → Google Client ID |
| **Google Client Secret** | Google Cloud Console → OAuth 클라이언트 생성 시 함께 발급 (5단계) | `GOCSPX-xxxxxx` | 환경설정 → Google Client Secret |

> 💡 **팁**: 위 표를 체크리스트처럼 활용하세요! 각 단계를 진행하면서 값을 메모장에 하나씩 복사해 두면 7단계에서 한 번에 입력할 수 있습니다.

---

## 1단계: Gemini API Key 발급

AI가 블로그 글을 생성하려면 Gemini API Key가 필요합니다.

### 진행 방법

1. **Google AI Studio** 접속  
   👉 [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey)

2. Google 계정으로 **로그인**

3. **[API 키 만들기]** 또는 **[Create API Key]** 버튼 클릭

4. 프로젝트를 선택하라는 창이 나오면 → **기존 프로젝트 아무거나 선택** 또는 **새 프로젝트 생성**

5. 생성된 API Key를 **복사** (예: `AIzaSy...xxxxx`)

6. 프로그램의 **환경설정 탭** → **Gemini API** 필드에 붙여넣기

> ⚠️ **이미지 생성까지 하려면?**  
> Google AI Studio에서 '유료 티어'(결제 수단 등록)로 전환해야 이미지 생성 API가 활성화됩니다.  
> 글 생성만 할 경우에는 무료 티어로도 충분합니다.

---

## 2단계: Google Cloud Console — 새 프로젝트 생성

블로그에 자동으로 글을 올리려면 Google로부터 '통행증'(OAuth 인증)을 받아야 합니다.  
이 과정은 **한 번만** 하면 됩니다.

### 진행 방법

1. **Google Cloud Console** 접속  
   👉 [https://console.cloud.google.com](https://console.cloud.google.com)

2. Google 계정으로 **로그인**

3. 화면 **상단 왼쪽**에 프로젝트 이름이 표시된 드롭다운을 클릭  
   (처음이면 "My First Project" 등이 보일 수 있습니다)

4. 팝업 창 **오른쪽 상단**의 **[새 프로젝트]** 클릭

5. 프로젝트 정보 입력:
   - **프로젝트 이름**: 원하는 이름 입력 (예: `MyBlogAuto` 또는 `블로그자동화`)
   - **조직**: 없음 (개인 계정이면 "조직 없음"으로 두세요)

6. **[만들기]** 클릭

7. 생성 완료 후, 상단 드롭다운에서 **방금 만든 프로젝트를 선택**하세요  
   (자동으로 선택되지 않을 수 있습니다. 반드시 확인!)

---

## 3단계: Blogger API 활성화

Google이 블로그스팟에 글을 올리도록 허용하는 단계입니다.

### 진행 방법

1. 상단 **검색창**에 `Blogger API` 입력

2. 검색 결과에서 **"Blogger API"** 클릭  
   (⚠️ "Blogger API v3"가 맞습니다. 다른 유사 이름 주의)

3. **[사용]** 버튼 클릭

4. "API가 사용 설정되었습니다" 메시지가 나타나면 성공! ✅

> 💡 만약 "사용" 버튼이 안 보이고 "관리"가 보이면 → 이미 활성화된 상태입니다.

---

## 4단계: OAuth 동의 화면 설정

사용자(본인)에게 "이 프로그램이 내 블로그에 접근해도 괜찮은지" 묻는 화면을 설정하는 단계입니다.

### 진행 방법

1. 좌측 메뉴에서 **[API 및 서비스]** → **[OAuth 동의 화면]** 클릭  
   (또는 상단 검색창에 "OAuth 동의 화면" 검색)

2. **User Type** 선택 화면:
   - ✅ **[외부]** 선택 (개인 Gmail 계정이면 무조건 "외부")
   - [만들기] 클릭

3. **앱 정보** 입력:
   | 항목 | 입력값 |
   |------|--------|
   | 앱 이름 | 아무 이름 (예: `블로그 자동화`) |
   | 사용자 지원 이메일 | 본인 Gmail 주소 선택 |
   | 개발자 연락처 정보 | 본인 Gmail 주소 입력 |

4. 나머지 항목은 **비워두고** → **[저장 후 계속]** 클릭

5. **범위(Scopes)** 화면 → 아무것도 안 해도 됨 → **[저장 후 계속]** 클릭

6. ⚠️ **테스트 사용자** 화면 (매우 중요!):
   - **[+ ADD USERS]** 클릭
   - 본인 **Gmail 주소** 입력 (예: `myname@gmail.com`)
   - **[추가]** 클릭

   > 🚨 **이 단계를 빠뜨리면 나중에 인증 시 "이 앱은 Google에서 확인하지 않았습니다" 오류가 나고 진행이 안 됩니다!**

7. **[저장 후 계속]** → 요약 화면에서 **[대시보드로 돌아가기]** 클릭

### (선택) 프로덕션 게시

> 테스트 모드에서는 인증 토큰이 **7일마다 만료**됩니다.  
> 매번 재인증하기 싫으면 아래처럼 프로덕션으로 전환하세요.

1. OAuth 동의 화면 → **[앱 게시]** 버튼 클릭
2. 확인 팝업에서 **[확인]** 클릭
3. 게시 상태가 "테스트" → **"프로덕션"**으로 변경되면 완료

> 💡 프로덕션 전환 시 Google 검토가 필요할 수 있지만, 본인만 사용하는 앱이라면 보통 바로 전환됩니다.

---

## 5단계: OAuth 클라이언트 ID & Secret 발급

프로그램이 Google에 "나 인증된 앱이야"라고 증명하기 위한 **신분증**을 만드는 단계입니다.

### 진행 방법

1. 좌측 메뉴에서 **[API 및 서비스]** → **[사용자 인증 정보]** 클릭

2. 상단의 **[+ 사용자 인증 정보 만들기]** 클릭

3. **[OAuth 클라이언트 ID]** 클릭

4. 설정 입력:
   | 항목 | 선택/입력값 |
   |------|-------------|
   | 애플리케이션 유형 | **데스크톱 앱** |
   | 이름 | 아무 이름 (예: `블로그자동화앱`) |

5. **[만들기]** 클릭

6. 🎉 팝업 창에 **두 가지 값이 표시됩니다**:

   | 항목 | 예시 | 어디에 입력? |
   |------|------|-------------|
   | **클라이언트 ID** | `123456789-xxxxxxx.apps.googleusercontent.com` | 프로그램의 "Google Client ID" 필드 |
   | **클라이언트 보안 비밀번호** | `GOCSPX-xxxxxxxxxxxxxx` | 프로그램의 "Google Client Secret" 필드 |

7. 두 값을 **메모장에 복사해두세요** (팝업 닫으면 Secret은 다시 볼 수 없음!)

> ⚠️ 팝업을 그냥 닫았다면?  
> [사용자 인증 정보] 페이지 → 만든 OAuth 클라이언트 이름 클릭 → Client ID는 보임, Secret은 새로 생성 필요

---

## 6단계: Blog ID 확인

블로그스팟에서 "어떤 블로그에 글을 올릴지" 지정하는 숫자 ID입니다.

### 진행 방법

1. **Blogger 대시보드** 접속  
   👉 [https://www.blogger.com](https://www.blogger.com)

2. 본인 블로그 선택 → **대시보드(관리 화면)** 진입

3. 브라우저 **주소창**을 확인하세요. 아래처럼 생겼습니다:

   ```
   https://www.blogger.com/blog/posts/1234567890123456789
   ```

4. `posts/` 뒤의 **긴 숫자**가 바로 Blog ID입니다!

   ```
   Blog ID = 1234567890123456789
   ```

5. 이 숫자를 **복사** 해두세요

> 💡 **블로그가 여러 개인 경우**  
> 각 블로그마다 Blog ID가 다릅니다. 자동 발행할 블로그의 대시보드에 들어가서 확인하세요.

---

## 7단계: 프로그램에 정보 입력 + 인증 완료

지금까지 수집한 정보를 프로그램에 입력하는 마지막 단계입니다.

### 진행 방법

1. 프로그램 실행 → **환경설정** 탭 이동

2. 플랫폼 선택: **Blogger** 라디오 버튼 클릭  
   (주황색 Blogger 설정 영역이 나타납니다)

3. 아래 3개 필드에 정보 입력:

   | 필드명 | 입력 값 | 예시 |
   |--------|---------|------|
   | **Blog ID** | 6단계에서 복사한 숫자 | `1234567890123456789` |
   | **Google Client ID** | 5단계에서 복사한 Client ID | `123456-xxx.apps.googleusercontent.com` |
   | **Google Client Secret** | 5단계에서 복사한 Secret | `GOCSPX-xxxxxx` |

4. **[🚀 인증하기]** 버튼 클릭

5. 브라우저 창이 자동으로 열립니다:
   - Google 계정 선택 화면 → **본인 계정 클릭**
   - "이 앱은 Google에서 확인하지 않았습니다" 경고가 나오면:
     1. 왼쪽 하단 **[고급]** 클릭
     2. **["(앱 이름)"(으)로 이동(안전하지 않음)]** 클릭
   - 권한 허용 화면 → **[계속]** 또는 **[허용]** 클릭

6. "✅ Blogger 인증이 완료되었습니다!" 메시지가 나타나면 **성공!** 🎉

> 인증 완료 후에는 토큰이 자동 저장되므로 매번 인증할 필요 없습니다.  
> (단, 테스트 모드일 때는 7일마다 재인증 필요 → 4단계의 "프로덕션 게시" 참고)

---

## ✅ 설정 완료 후 테스트

1. **포스팅 탭** 이동
2. 키워드 입력 (예: `블로그 수익화 방법`)
3. **[🚀 블로그 글 작성 시작]** 클릭
4. 로그에 "✅ 발행 완료" 메시지 확인
5. 블로그스팟 대시보드에서 실제로 글이 올라갔는지 확인

---

## 🚨 자주 겪는 문제 & 해결법

### ❌ "이 앱은 Google에서 확인하지 않았습니다" 오류
**원인**: OAuth 동의 화면에서 테스트 사용자를 추가하지 않음  
**해결**: 4단계-6번 참고하여 본인 Gmail을 테스트 사용자에 추가

### ❌ "redirect_uri_mismatch" 오류
**원인**: 애플리케이션 유형을 "데스크톱 앱"이 아닌 다른 것으로 선택함  
**해결**: 5단계에서 반드시 **"데스크톱 앱"** 선택 후 재생성

### ❌ 인증은 됐는데 7일마다 만료
**원인**: OAuth 동의 화면이 테스트 모드  
**해결**: 4단계 하단의 "프로덕션 게시"를 진행

### ❌ "Blogger API has not been used" 오류
**원인**: Blogger API를 활성화하지 않음  
**해결**: 3단계 참고하여 Blogger API **[사용]** 클릭

### ❌ "Blog ID를 찾을 수 없습니다" 오류
**원인**: Blog ID를 잘못 입력함 (URL 전체를 넣거나 다른 숫자 넣음)  
**해결**: 6단계 참고하여 `posts/` 뒤의 숫자만 정확히 입력

### ❌ Client ID / Secret이 기억나지 않음
**해결**: [Google Cloud Console → 사용자 인증 정보](https://console.cloud.google.com/apis/credentials) 에서 다시 확인 가능  
(Client Secret은 재확인 불가 → 새로 생성 필요)
