# 핵심요약표 구조 분석 및 개선 제안

## 현재 구조

### 테이블 레이아웃
```
┌──────────┬──────────────────────────────┬──────────┐
│ 소제목   │ 핵심내용                      │ 관련링크 │
│ (20%)    │ (60%)                        │ (20%)    │
├──────────┼──────────────────────────────┼──────────┤
│ 1번 소제목│ [섹션 제목]                  │ [링크]   │
│          │ [요약 내용]                   │          │
├──────────┼──────────────────────────────┼──────────┤
│ 2번 소제목│ [섹션 제목]                  │ [링크]   │
│          │ [요약 내용]                   │          │
└──────────┴──────────────────────────────┴──────────┘
```

### 현재 데이터 구조
- **첫 번째 컬럼 (`<th>`)**: "1번 소제목", "2번 소제목" 등 (라벨만 표시)
- **두 번째 컬럼 (`<td>`)**: 
  - 상단: 섹션 제목 (진한 검은색, 굵게)
  - 하단: 요약 내용 (검은색, 일반)
- **세 번째 컬럼 (`<td>`)**: 관련 링크 또는 "준비 중"

## 문제점 분석

### 1. 정보 구조의 혼란
- 첫 번째 컬럼에 "1번 소제목"이라는 라벨만 있어서 실제 정보가 없음
- 섹션 제목이 두 번째 컬럼에 들어가 있어서 헤더와 불일치
- 독자가 "소제목" 컬럼을 봤을 때 실제 소제목을 찾기 어려움

### 2. 가독성 문제
- 섹션 제목과 요약이 같은 컬럼에 있어서 시각적 구분이 약함
- 첫 번째 컬럼이 20%로 넓은데 실제 내용은 라벨만 있어서 공간 낭비

### 3. 정보 밀도
- 핵심 정보(섹션 제목, 요약, 링크)는 있지만 구조가 직관적이지 않음

## 개선 제안

### 옵션 1: 현재 구조 유지 + 첫 번째 컬럼에 실제 섹션 제목 표시
```
┌──────────────────┬──────────────────────────────┬──────────┐
│ 소제목           │ 핵심내용                      │ 관련링크 │
│ (30%)            │ (50%)                        │ (20%)    │
├──────────────────┼──────────────────────────────┼──────────┤
│ [실제 섹션 제목] │ [요약 내용]                  │ [링크]   │
│ (진한 검은색)    │ (검은색)                     │          │
├──────────────────┼──────────────────────────────┼──────────┤
│ [실제 섹션 제목] │ [요약 내용]                  │ [링크]   │
└──────────────────┴──────────────────────────────┴──────────┘
```

**장점:**
- 헤더와 데이터가 일치
- 섹션 제목이 명확하게 보임
- 구조가 직관적

**단점:**
- 섹션 제목이 길면 첫 번째 컬럼이 좁아 보일 수 있음

### 옵션 2: 2컬럼 구조로 단순화
```
┌──────────────────────────────┬──────────┐
│ 핵심내용                      │ 관련링크 │
│ (80%)                        │ (20%)    │
├──────────────────────────────┼──────────┤
│ [섹션 제목]                   │ [링크]   │
│ [요약 내용]                   │          │
├──────────────────────────────┼──────────┤
│ [섹션 제목]                   │ [링크]   │
│ [요약 내용]                   │          │
└──────────────────────────────┴──────────┘
```

**장점:**
- 구조가 단순하고 명확
- 핵심 정보에 집중
- 모바일에서도 읽기 쉬움

**단점:**
- "소제목"이라는 헤더가 없어서 구조적 정보가 약함

### 옵션 3: 현재 구조 개선 (추천)
```
┌──────────────────┬──────────────────────────────┬──────────┐
│ 소제목           │ 핵심내용                      │ 관련링크 │
│ (25%)            │ (55%)                        │ (20%)    │
├──────────────────┼──────────────────────────────┼──────────┤
│ [실제 섹션 제목] │ [요약 내용]                  │ [링크]   │
│ (진한 검은색)    │ (검은색, 2-3줄)              │          │
│ (굵게, 큰 글씨)  │                              │          │
├──────────────────┼──────────────────────────────┼──────────┤
│ [실제 섹션 제목] │ [요약 내용]                  │ [링크]   │
└──────────────────┴──────────────────────────────┴──────────┘
```

**개선 사항:**
1. 첫 번째 컬럼에 실제 섹션 제목 표시 (번호 제거)
2. 두 번째 컬럼에는 요약 내용만 표시
3. 컬럼 너비 조정: 25% / 55% / 20%
4. 섹션 제목을 더 크고 굵게 표시하여 가독성 향상

## 현재 스타일 특징

### 시각적 디자인
- ✅ 흰색 배경, 파란색 그라데이션 헤더
- ✅ 둥근 모서리 (28px)
- ✅ 부드러운 그림자 효과
- ✅ 반응형 디자인 (모바일 최적화)
- ✅ 짝수/홀수 행 배경색 구분 (가독성 향상)

### 텍스트 색상
- 제목: 진한 검은색 (#1a1a1a)
- 본문: 검은색 (#000000)
- 링크: 파란색 (#2563eb)

### 반응형
- 768px 이하: 패딩 축소, 폰트 크기 축소
- 480px 이하: 더 작은 패딩과 폰트 크기

## 결론

현재 구조는 **시각적으로는 아름답지만 정보 구조가 직관적이지 않습니다**. 

**추천 개선안:**
- 첫 번째 컬럼에 실제 섹션 제목을 표시하여 헤더와 일치시키기
- "1번 소제목" 같은 라벨 제거
- 섹션 제목을 더 크고 굵게 표시하여 한눈에 들어오도록 개선

이렇게 하면 독자가 테이블을 보는 순간 각 행이 어떤 섹션인지 바로 알 수 있고, 요약 내용과 링크를 빠르게 파악할 수 있습니다.
