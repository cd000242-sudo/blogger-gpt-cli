# 글 스타일 적용 방식

## 현재 구조

### 1. AI 생성 단계
- **프롬프트**: 스타일 없이 순수 HTML만 생성하도록 지시
- AI가 생성하는 HTML: `<h2>제목</h2>`, `<h3>소제목</h3>` (스타일 없음)

### 2. 후처리 단계 (섹션 단계)
**위치**: `src/core/index.ts` 3751-3802번 라인

#### H2 스타일 (3754번 라인)
```css
background: linear-gradient(135deg, rgba(16, 185, 129, 0.18) 0%, rgba(5, 150, 105, 0.08) 100%);
color: #064e3b;
font-size: clamp(1.75rem, 2.5vw, 2.25rem);
font-weight: 700;
padding: 20px 24px;
margin: 40px 0 24px 0;
border-radius: 12px;
border-left: 4px solid #10b981;
```
**결과**: 녹색 그라데이션 배경, 녹색 텍스트, 왼쪽 녹색 테두리

#### H3 스타일 (3787번 라인)
```css
background: linear-gradient(135deg, rgba(59, 130, 246, 0.12) 0%, rgba(37, 99, 235, 0.08) 100%);
color: #1e40af;
font-size: clamp(1.5rem, 2vw, 1.875rem);
font-weight: 600;
padding: 16px 20px;
margin: 32px 0 20px 0;
border-radius: 10px;
border-left: 3px solid #3b82f6;
```
**결과**: 파란색 그라데이션 배경, 파란색 텍스트, 왼쪽 파란색 테두리

#### P (첫 번째 문단) 스타일 (3764번 라인)
```css
font-size: 22px;
line-height: 1.9;
color: #2c3e50;
margin: 0 0 28px 0;
font-weight: 400;
```

### 3. 최종 후처리 단계
**위치**: `src/core/index.ts` 5689-5731번 라인

- 섹션 단계에서 적용한 스타일을 한 번 더 강제 적용
- `fullHtml` 생성 직전에 최종 안전장치로 실행

### 4. CSS 스타일 (전역)
**위치**: `src/core/index.ts` 4311번 라인 (cloudSkinCSS) 또는 5134번 라인 (premiumGradientCSS)

- 전체 글의 기본 스타일 정의
- 인라인 스타일과 함께 적용됨

## 최종 결과

### H2 제목
- **배경**: 연한 녹색 그라데이션
- **텍스트**: 진한 녹색 (#064e3b)
- **크기**: 반응형 (1.75rem ~ 2.25rem)
- **테두리**: 왼쪽에 4px 녹색 테두리
- **모서리**: 둥근 모서리 (12px)

### H3 소제목
- **배경**: 연한 파란색 그라데이션
- **텍스트**: 진한 파란색 (#1e40af)
- **크기**: 반응형 (1.5rem ~ 1.875rem)
- **테두리**: 왼쪽에 3px 파란색 테두리
- **모서리**: 둥근 모서리 (10px)

### 본문 문단
- **크기**: 22px
- **줄간격**: 1.9
- **색상**: 진한 회색 (#2c3e50)
- **여백**: 하단 28px

## 적용 순서

1. AI가 스타일 없이 HTML 생성
2. 섹션 단계에서 모든 H2/H3에 인라인 스타일 강제 적용
3. 최종 후처리에서 한 번 더 강제 적용 (안전장치)
4. CSS와 함께 최종 HTML 생성

## 장점

- ✅ AI가 생성한 스타일과 무관하게 통일된 디자인
- ✅ 반응형 디자인 (모바일/데스크톱 자동 조정)
- ✅ 접근성 향상 (충분한 대비, 읽기 쉬운 크기)
- ✅ 일관된 브랜드 스타일
