# 라이선스 인증 UI 방식 추천

## 현재 상황 분석

### 현재 방식 (라이선스 인증 버튼)
- ✅ **장점**:
  - 필요할 때만 인증 (선택적)
  - 사용자가 원하는 시점에 인증 가능
  - UI가 깔끔하고 간단함
  - 이미 구현되어 있음

- ❌ **단점**:
  - 사용자가 인증을 잊을 수 있음
  - 앱 시작 시 자동 인증 불가

### 로그인 창 방식
- ✅ **장점**:
  - 앱 시작 시 자동으로 인증 요구
  - 인증 상태가 명확함
  - 보안성이 높음 (매번 확인)

- ❌ **단점**:
  - 매번 입력해야 하는 불편함
  - 사용자 경험이 다소 번거로울 수 있음
  - 개발 시간이 더 필요함

## 🎯 추천 방식: **하이브리드 방식**

### 구현 방안

1. **앱 시작 시 자동 인증 확인**
   - 저장된 인증 정보가 있고 유효한 경우 → 자동 로그인
   - 저장된 정보가 없거나 만료된 경우 → 로그인 창 표시

2. **로그인 창 기능**
   - ID/비밀번호 입력
   - 라이선스 코드 입력 (선택사항)
   - "자동 로그인" 체크박스
   - 기간제: 코드 기간이 남으면 자동 로그인 가능
   - 영구제: 패치 파일 있으면 자동 로그인 가능

3. **기존 라이선스 인증 버튼 유지**
   - 수동으로 재인증하고 싶을 때 사용
   - 설정에서 언제든지 재인증 가능

### 사용자 경험 플로우

```
앱 시작
  ↓
저장된 인증 정보 확인
  ↓
유효한가? ──Yes──> 자동 로그인 → 앱 사용
  ↓ No
로그인 창 표시
  ↓
ID/비밀번호 입력
  ↓
코드 입력 (선택)
  ↓
인증 성공
  ↓
"자동 로그인" 체크 → 저장 → 다음부터 자동 로그인
```

## 💡 최종 추천

**하이브리드 방식**을 추천합니다:

1. **앱 시작 시**: 저장된 인증 정보로 자동 로그인 시도
2. **자동 로그인 실패 시**: 로그인 창 표시
3. **설정에서**: 언제든지 라이선스 인증 버튼으로 재인증 가능

이 방식의 장점:
- ✅ 사용자 편의성 (자동 로그인)
- ✅ 보안성 (필요 시 재인증)
- ✅ 유연성 (선택적 인증)
- ✅ 기존 기능 유지

## 구현 우선순위

1. **1단계**: 네이버 접속 제한 우회 구현 ✅ (완료)
2. **2단계**: 자동 로그인 기능 추가 (저장된 인증 정보 확인)
3. **3단계**: 로그인 창 UI 구현 (앱 시작 시 표시)
4. **4단계**: 기존 라이선스 인증 버튼과 통합
