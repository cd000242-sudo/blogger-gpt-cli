# 블로그스팟 발행 문제 해결 가이드

블로그스팟 발행이 안 될 때 다음 단계를 순서대로 확인해주세요.

## 1. 인증 상태 확인

### 증상
- "토큰 파일이 없습니다" 오류
- "인증이 필요합니다" 오류
- "토큰이 만료되었습니다" 오류

### 해결 방법
1. **환경설정 탭**으로 이동
2. **"Blogger OAuth2 인증"** 버튼 클릭
3. 브라우저가 열리면 **Google 계정으로 로그인**
4. 권한 승인 화면에서 **"허용"** 클릭
5. 생성된 **인증 코드를 복사**
6. 앱으로 돌아와서 코드를 **입력란에 붙여넣기**
7. **"인증 완료"** 버튼 클릭

## 2. Google OAuth2 설정 확인

### 증상
- "Google OAuth2 Client ID/Secret이 누락되었습니다" 오류

### 해결 방법

#### Google Cloud Console에서 OAuth 2.0 클라이언트 ID 생성

1. **Google Cloud Console** 접속
   - https://console.cloud.google.com

2. **프로젝트 생성** (없는 경우)
   - 상단의 프로젝트 선택 > 새 프로젝트
   - 프로젝트 이름 입력 (예: "blogger-gpt-cli")

3. **Blogger API 활성화**
   - 좌측 메뉴 > API 및 서비스 > 라이브러리
   - "Blogger API" 검색
   - "사용 설정" 클릭

4. **OAuth 동의 화면 구성**
   - 좌측 메뉴 > API 및 서비스 > OAuth 동의 화면
   - 사용자 유형: "외부" 선택 > "만들기"
   - 앱 이름: "Blogger GPT CLI" (또는 원하는 이름)
   - 사용자 지원 이메일: 본인 이메일
   - 개발자 연락처 정보: 본인 이메일
   - "저장 후 계속" 클릭
   - 범위 추가: "저장 후 계속" 클릭 (기본값 사용)
   - 테스트 사용자 추가: 본인 Gmail 주소 추가
   - "저장 후 계속" 클릭

5. **OAuth 2.0 클라이언트 ID 생성**
   - 좌측 메뉴 > API 및 서비스 > 사용자 인증 정보
   - 상단의 "+ 사용자 인증 정보 만들기" > "OAuth 클라이언트 ID"
   - 애플리케이션 유형: "데스크톱 앱" 선택
   - 이름: "Blogger GPT CLI Desktop" (또는 원하는 이름)
   - "만들기" 클릭

6. **클라이언트 ID와 Secret 복사**
   - 생성된 클라이언트 ID 복사
   - 클라이언트 보안 비밀번호(Secret) 복사

7. **앱 환경설정에 입력**
   - 환경설정 탭으로 이동
   - Google Client ID 입력란에 붙여넣기
   - Google Client Secret 입력란에 붙여넣기
   - "저장" 클릭

## 3. Blog ID 확인

### 증상
- "Blogger Blog ID가 비어 있습니다" 오류
- "Blog ID가 올바른 형식이 아닙니다" 오류

### Blog ID 찾는 방법

#### 방법 1: 블로그 설정에서 확인
1. **Blogger.com** 접속 (https://www.blogger.com)
2. 블로그 선택
3. 좌측 메뉴 > **설정** > **기본**
4. 페이지 상단의 URL 확인
   - 예: `https://www.blogger.com/blog/posts/1234567890123456789`
   - 여기서 `1234567890123456789`가 Blog ID

#### 방법 2: 블로그 HTML 소스에서 확인
1. 본인의 블로그 방문 (예: https://example.blogspot.com)
2. 페이지 소스 보기 (Ctrl+U 또는 우클릭 > 페이지 소스 보기)
3. `blogId` 검색
4. `"blogId":"1234567890123456789"` 형태로 표시됨

#### 방법 3: Blogger API Explorer 사용
1. https://developers.google.com/blogger/docs/3.0/reference/blogs/listByUser 접속
2. "Try this API" 섹션에서 "Execute" 클릭
3. 본인 블로그 목록에서 `id` 필드 확인

### Blog ID 입력
- 환경설정 탭으로 이동
- Blog ID 입력란에 **숫자만** 입력 (하이픈이나 공백 없이)
- 예: `1234567890123456789`
- "저장" 클릭

## 4. 권한 확인

### 증상
- "Blogger 권한이 부족합니다" 오류 (403 에러)

### 해결 방법
1. **블로그 소유권 확인**
   - 본인이 해당 블로그의 소유자 또는 작성자인지 확인
   - Blogger.com에서 블로그 목록에 표시되는지 확인

2. **Google 계정 확인**
   - 인증 시 사용한 Google 계정과 블로그 소유자 계정이 동일한지 확인

3. **재인증**
   - 환경설정 > "Blogger OAuth2 인증" 다시 진행
   - 올바른 Google 계정으로 로그인했는지 확인

## 5. API 할당량 확인

### 증상
- "Google Blogger API 일일 할당량을 초과했습니다" 오류

### 해결 방법
1. **내일 다시 시도**
   - Google Blogger API는 일일 할당량이 있습니다
   - 자정(UTC 기준)에 할당량이 리셋됩니다

2. **할당량 늘리기** (필요한 경우)
   - Google Cloud Console 접속
   - API 및 서비스 > 할당량
   - Blogger API 할당량 확인 및 증가 요청

## 6. 콘텐츠 크기 확인

### 증상
- "콘텐츠 크기가 너무 큽니다" 오류

### 해결 방법
- Blogger API는 포스트당 약 **200KB** 제한이 있습니다
- 다음 방법으로 크기 줄이기:
  1. 이미지 수 줄이기
  2. 이미지 크기 최적화
  3. 불필요한 CSS 제거
  4. 본문 내용 축소

## 7. 네트워크 연결 확인

### 증상
- "네트워크 오류" 또는 "타임아웃" 오류

### 해결 방법
1. **인터넷 연결 확인**
   - 브라우저에서 Google.com 접속 가능한지 확인

2. **방화벽/프록시 확인**
   - 회사 네트워크나 방화벽이 Google API 접근을 차단하는지 확인
   - VPN 사용 중이라면 VPN 끄고 다시 시도

3. **DNS 확인**
   - DNS 서버가 정상 작동하는지 확인
   - Google DNS (8.8.8.8) 사용 시도

## 8. 로그 확인

### 디버깅을 위한 로그 확인 방법

1. **앱 로그 확인**
   - 발행 시 하단의 로그 창 확인
   - 빨간색 오류 메시지 확인

2. **콘솔 로그 확인** (개발자용)
   - Windows: Ctrl+Shift+I
   - Mac: Cmd+Option+I
   - Console 탭에서 `[PUBLISH]` 또는 `[AUTH]` 로그 확인

3. **토큰 파일 확인**
   - Windows: `%APPDATA%\blogger-gpt-cli\blogger-token.json`
   - Mac: `~/Library/Application Support/blogger-gpt-cli/blogger-token.json`
   - Linux: `~/.config/blogger-gpt-cli/blogger-token.json`

## 9. 일반적인 해결 방법

### 모든 설정 초기화 후 재설정

1. **앱 종료**

2. **설정 파일 삭제**
   - Windows: `%APPDATA%\blogger-gpt-cli` 폴더 삭제
   - Mac: `~/Library/Application Support/blogger-gpt-cli` 폴더 삭제
   - Linux: `~/.config/blogger-gpt-cli` 폴더 삭제

3. **앱 재시작**

4. **처음부터 설정**
   - Google OAuth2 설정 (2번 참조)
   - Blog ID 입력 (3번 참조)
   - Blogger OAuth2 인증 (1번 참조)

## 10. 여전히 문제가 해결되지 않는 경우

### 추가 지원

1. **GitHub Issues**
   - 프로젝트 GitHub 페이지에서 Issue 생성
   - 다음 정보 포함:
     - 오류 메시지 전체
     - 발생 시점
     - 시도한 해결 방법
     - 로그 파일 (민감한 정보 제거 후)

2. **일반적인 체크리스트**
   - [ ] Google OAuth2 Client ID/Secret 설정됨
   - [ ] Blog ID가 올바르게 입력됨 (숫자만)
   - [ ] Blogger OAuth2 인증 완료됨
   - [ ] 인증 시 사용한 계정이 블로그 소유자임
   - [ ] Blogger API가 Google Cloud Console에서 활성화됨
   - [ ] 인터넷 연결이 정상임
   - [ ] 방화벽이 Google API를 차단하지 않음
